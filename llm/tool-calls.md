`tool_calls` 是 OpenAI 在 Chat Completion API 中引入的一种机制，允许模型在生成回复时请求调用一个或多个外部工具（如函数、API）。它的生成涉及模型训练、推理过程中的特殊输出格式，以及服务端对模型输出的解析与结构化。下面通过一个天气查询的例子，逐步说明其机制与原理。

---

## 1. 背景：工具调用的需求

在复杂对话中，模型可能无法直接回答某些问题（例如查询实时天气），需要借助外部工具获取信息。OpenAI 通过 `tools` 参数（或在旧版中使用 `functions`）让开发者描述可用的工具。当模型决定使用工具时，它会输出结构化的调用请求，而非纯文本。

---

## 2. 生成 `tool_calls` 的完整流程

### 步骤 1：用户请求包含工具定义
客户端发送以下请求（简化）：
```json
{
  "model": "gpt-4o",
  "messages": [
    {"role": "user", "content": "北京今天天气怎么样？"}
  ],
  "tools": [
    {
      "type": "function",
      "function": {
        "name": "get_weather",
        "description": "获取指定城市的天气",
        "parameters": {
          "type": "object",
          "properties": {
            "location": {"type": "string"}
          },
          "required": ["location"]
        }
      }
    }
  ]
}
```

### 步骤 2：模型内部决策与生成
- **理解上下文**：模型根据对话历史（用户问题）和提供的工具列表，判断需要调用 `get_weather` 来获取实时数据。
- **特殊训练**：模型在训练阶段被教会了一种特殊的输出格式——当需要调用工具时，它不再生成普通文本，而是生成一个 **结构化的 JSON 对象**（或一系列对象）。这个 JSON 包含工具名称和参数，并且通常用特殊的标记包裹，以便服务端识别。
- **自回归生成**：模型像往常一样逐个预测 token，但此时它输出的 token 序列不再是自然语言，而是类似以下内容：
  ```
  {"name": "get_weather", "arguments": {"location": "北京"}}
  ```
  如果启用了并行工具调用，模型可能会生成多个这样的 JSON 对象，例如：
  ```
  [{"id": "call1", "type": "function", "function": {"name": "get_weather", "arguments": {"location": "北京"}}}, {"id": "call2", ...}]
  ```
  注意，实际的底层 token 可能包含特殊标记（如 `<|tool_call|>`），但最终呈现给客户端的 API 响应会剥离这些标记，只保留结构化数据。

### 步骤 3：服务端解析与封装
- **检测工具调用**：在生成过程中，服务端实时监测输出的 token 序列。一旦检测到符合工具调用格式的内容，它会将该部分提取出来，并从 `content` 字段中移除（因此最终响应中 `content` 通常为 `null` 或不包含工具调用的文本）。
- **构造 `tool_calls` 数组**：服务端将解析出的一个或多个工具调用放入 `tool_calls` 数组，每个调用包含：
  - `id`：一个唯一标识符，用于后续将工具执行结果与本次调用关联。
  - `type`：目前固定为 `"function"`。
  - `function`：包含 `name` 和 `arguments`（一个 JSON 字符串）。
- **设置 `finish_reason`**：服务端将 `finish_reason` 设为 `"tool_calls"`，表明生成结束是因为模型发出了工具调用请求，而非自然结束或长度限制。

### 步骤 4：最终返回的 `ChatCompletion` 响应
```json
{
  "id": "chatcmpl-abc123",
  "object": "chat.completion",
  "created": 1699987654,
  "model": "gpt-4o",
  "choices": [
    {
      "index": 0,
      "finish_reason": "tool_calls",
      "message": {
        "role": "assistant",
        "content": null,
        "tool_calls": [
          {
            "id": "call_abc123",
            "type": "function",
            "function": {
              "name": "get_weather",
              "arguments": "{\"location\":\"北京\"}"
            }
          }
        ]
      }
    }
  ],
  "usage": {
    "prompt_tokens": 25,
    "completion_tokens": 18,
    "total_tokens": 43
  }
}
```

---

## 3. 原理详解

### 3.1 模型训练层面
- **训练数据**：OpenAI 在训练模型时，构建了大量包含工具调用示例的数据。这些示例中，模型在适当的位置输出一个特殊的标记序列（例如 `"<|tool_call|>"`），后面跟着 JSON 格式的工具调用。模型学会了在需要外部信息时生成这种模式。
- **指令微调**：通过指令微调（instruction tuning），模型进一步明确了何时应该调用工具、如何构造参数 JSON。

### 3.2 推理与服务端处理
- **约束解码（可选）**：服务端可能使用约束解码技术，强制模型在特定上下文中只生成符合工具调用格式的 token，确保输出是有效的 JSON。
- **检测与分割**：服务端在流式生成过程中，会识别特殊标记（如开始调用标记和结束调用标记），将工具调用部分与普通文本分离。对于非流式响应，服务端在生成完成后对整个 token 序列进行后处理，提取工具调用。
- **结构化转换**：将提取的 JSON 字符串解析为对象，并分配唯一的 `id`。这允许客户端并发执行多个工具调用，并正确地将结果返回给模型。

### 3.3 并行工具调用
新版本支持在一个响应中包含多个工具调用，这通过在工具定义中设置 `parallel_tool_calls: true`（默认为 true）实现。模型可以生成一个 JSON 数组，包含多个调用。服务端将其拆分为独立的 `tool_calls` 数组元素，每个元素都有独立的 `id`。这样客户端可以并行执行这些调用，提高效率。

---

## 4. 为什么需要 `tool_calls` 而不是直接在 `content` 里放 JSON？

- **清晰的角色分离**：`content` 字段保留给纯文本回复，而 `tool_calls` 专门用于表示工具调用。这避免了模型输出与工具调用混合时的解析歧义。
- **多调用支持**：`tool_calls` 数组允许模型在一次响应中请求多个工具调用，而 `function_call`（旧版）只能支持一个。
- **结果关联**：每个工具调用都有唯一的 `id`，客户端执行完工具后，可以通过 `tool_call_id` 将结果精确地关联回原始调用，然后以 `tool` 角色消息返回给模型。

---

## 5. 与旧版 `function_call` 的对比

| 特性 | `function_call` (旧) | `tool_calls` (新) |
|------|----------------------|-------------------|
| 支持数量 | 单个 | 多个 |
| 结果关联 | 无独立 ID，需靠顺序匹配 | 每个调用有独立 `id` |
| 字段位置 | 消息根级别 | 消息内 `tool_calls` 数组 |
| 扩展性 | 仅限函数 | 可扩展至其他工具类型（目前仍是 function） |

---

## 6. 总结

- **机制**：`tool_calls` 是模型在特定训练下学会的一种输出模式，通过生成结构化的 JSON 表示工具调用请求。
- **原理**：服务端在生成过程中检测、提取这些结构，并将其从 `content` 中分离，封装为 `tool_calls` 数组，同时设置 `finish_reason` 为 `"tool_calls"`。
- **作用**：使模型能够与外部世界交互，获取实时信息或执行操作，并支持并发调用，大大扩展了 AI 应用的能力边界。

理解这一机制有助于开发者正确构建支持工具调用的应用，处理模型返回的调用请求，并将执行结果反馈给模型以完成多轮对话。
