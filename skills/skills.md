**翻译：**

Agent Skills 是一种轻量级、开放的格式，用于通过专业知识和 workflows（工作流程）扩展 AI 代理的能力。

其核心是，一个 skill（技能）就是一个包含 `SKILL.md` 文件的文件夹。该文件包含元数据（至少包括名称和描述）以及指导代理如何执行特定任务的指令。技能还可以捆绑脚本、模板和参考资料。

---

**详细解释：**

这段文字介绍了一个名为 "Agent Skills" 的概念。我们可以从以下几个方面来深入理解：

### 1. 什么是 Agent Skills？
Agent Skills 是一种**标准化、轻量级的封装方式**，旨在让 AI 代理（如聊天机器人、自动化助手等）能够轻松获得并执行特定领域的专业任务。它本质上是一种**插件化**或**技能化**的扩展机制。

*   **轻量级**：意味着它设计简单，易于创建、分享和使用，不需要复杂的 infrastructure。
*   **开放格式**：意味着它是一种公开的、非专有的标准，可以被不同的 AI 代理系统所兼容和采用。
*   **扩展 AI 代理能力**：AI 代理本身可能只有通用的对话和推理能力，而 Skills 为其注入了**专业知识和预设的工作流程**，使其能够完成更具体、更复杂的任务。

### 2. 核心组成：一个 Skill 就是一个文件夹
一个 Skill 被定义为一个文件夹，其中至少包含一个 `SKILL.md` 文件。这个文件夹可能还包含其他辅助文件。

*   **`SKILL.md` 文件**：这是技能的核心文档，采用 Markdown 格式编写。它主要包含两部分：
    *   **元数据 (Metadata)**：至少包括技能的名称和一段描述。描述告诉代理这个技能是做什么用的，在什么情况下应该调用它。这类似于工具调用中工具的定义（名称和描述）。
    *   **指令 (Instructions)**：这是**最关键的部分**。它用自然语言（或某种结构化格式）详细描述了代理应该**如何一步一步地执行这个任务**。这相当于给代理提供了一个操作手册或标准作业程序（SOP）。例如，一个“发送邮件”的技能，其指令可能包括：1. 获取收件人地址；2. 获取邮件主题和正文；3. 调用邮件 API；4. 处理发送结果等。

*   **可选的捆绑内容**：
    *   **脚本 (Scripts)**：可以是 Python、JavaScript 等可执行代码，用于实现具体的操作（比如调用 API、处理数据）。这类似于工具调用中的“工具代码”。
    *   **模板 (Templates)**：可以是常用的邮件模板、报告模板、代码片段等，代理可以直接使用或填充。
    *   **参考资料 (Reference Materials)**：可以是 PDF 文档、知识库文章、数据表等，代理在执行任务时可以查阅，以获取更准确的信息。

### 3. 与之前讨论的 "Tool Use" 有何关联与区别？
在之前的对话中，我们详细探讨了 Claude 的“工具调用 (Tool Use)”。这里提出的 "Agent Skills" 概念与其有相似之处，但范围和抽象层次可能不同。

*   **相似点**：
    *   **都是扩展 AI 能力**：两者都旨在让 AI 能做更多事情，超越基础模型的能力。
    *   **都有描述和输入**：工具调用有工具名称、描述和输入模式；Skills 也有元数据（名称、描述）和指令（可能包含输入要求）。
    *   **都可能涉及外部代码**：工具调用需要你在客户端运行代码；Skills 可以捆绑脚本，这些脚本就是被执行的代码。

*   **可能的区别**：
    *   **粒度与抽象层次**：工具调用通常更偏向于**单一、原子性的操作**（如：获取天气、计算股票）。而 Skills 可能是一个**更复杂的工作流程**，它可能内部会调用多个工具，或者结合了指令、模板和脚本，来完成一个更宏观的任务（如：“生成一份市场分析报告”，这个技能内部可能涉及数据查询、数据分析、图表生成、报告排版等多个步骤）。
    *   **标准化与开放性**："Agent Skills" 被描述为一种“开放格式”，旨在成为一种通用的、可跨平台使用的标准。而 Claude 的“工具调用”目前是其 API 的一种特定实现方式。Skills 的概念更抽象，可能旨在定义一种代理能力的交换格式，类似于 AI 领域的“插件标准”。
    *   **指令的详细程度**：Skills 的核心是 `SKILL.md` 中的**详细指令**，这些指令是给 AI 代理看的，指导它如何一步步思考和执行。而工具调用中，工具的描述是给模型看的，但具体的执行逻辑（代码）是写在客户端的。Skills 将部分“执行逻辑”也以指令形式放入了文件夹中，可能让代理有更大的自主决策空间。

### 4. 实际应用场景举例
假设我们有一个通用的 AI 代理，我们可以给它安装各种 Skills：

*   **“撰写周报”技能**：该技能包含一个周报模板（`.md` 文件）和一些指令，告诉代理如何从用户的日历、任务管理工具中提取本周完成的工作，然后填充到模板中，生成一份周报草稿。
*   **“预订会议室”技能**：该技能包含一个与公司会议室 API 交互的 Python 脚本，以及指令，告诉代理如何获取用户的时间要求、查询可用会议室、并调用脚本完成预订。
*   **“分析客户反馈”技能**：该技能包含一个情感分析脚本、一个关键词提取脚本，以及指令，指导代理如何读取客户评论 CSV 文件，运行分析，并生成一份包含情感趋势和热点问题的摘要报告。

### 总结
"Agent Skills" 提出了一种**模块化、可共享**的方式来构建 AI 代理的能力。它通过一个包含元数据、指令和可选资源的文件夹，将完成特定任务所需的知识和流程打包在一起。这使得 AI 代理的能力可以像搭积木一样被组合和扩展，也方便开发者社区共享和复用这些“技能”，从而加速 AI 应用的开发。它和“工具调用”在理念上一脉相承，但可能更侧重于**复杂任务的编排和知识的封装**，旨在成为一种更通用的代理扩展标准。
